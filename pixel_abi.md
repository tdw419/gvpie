# Pixel ABI v0.1

This document defines the Application Binary Interface (ABI) for the Pixel OS. All components, including the GVPIE Bootstrap (Rust/WGSL), the Pixel OS (Python), and any CUDA kernels, MUST adhere to this specification to ensure interoperability.

## 1. Coordinate System

*   **Origin**: Top-left corner `(0, 0)`.
*   **Axis**:
    *   `+X` is to the right.
    *   `+Y` is down.
*   **Units**: All coordinates are in pixels unless otherwise specified.

## 2. Textures

### 2.1. Machine Texture

*   **Format**: `RGBA8Uint` (one byte per channel, unsigned integer).
*   **Dimensions**: 128x64 pixels.
*   **Encoding**:
    *   **R Channel**: ASCII character code (0-127).
    *   **G Channel**: Unused (0).
    *   **B Channel**: Unused (0).
    *   **A Channel**: Unused (255).
*   **Purpose**: Compact representation of a program or text, sent from the OS to the GPU.

### 2.2. Human Texture

*   **Format**: `RGBA8Unorm` (one byte per channel, normalized float `0.0-1.0`).
*   **Dimensions**: 128x64 pixels.
*   **Encoding**: Grayscale glyphs expanded from the Machine Texture.
    *   **R, G, B Channels**: Grayscale value of the pixel (all three are identical).
    *   **A Channel**: Alpha (1.0 for pixels, 0.0 for background).
*   **Purpose**: Human-readable output from the GPU, for display or for recognition by the Pixel OS.

## 3. Glyphs and Fonts

*   **Glyph Size**: 5x7 pixels.
*   **Cell Size**: 8x8 pixels. Each glyph is rendered within an 8x8 cell.
*   **Glyph Atlas**: The canonical glyph data is defined below in a compact bitmask format. Each row is a `u8`, with the 5 least significant bits representing the pixels from left to right.

```

0x0E, 0x11, 0x11, 0x1F, 0x11, 0x11, 0x11
```
